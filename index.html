Computational Thinking: The thought processes involved in formulating a problem and expressing its solutions in such a way that a computer - human or machine - can effectively carry it out.

I would say computational thinking is about solving problems in an algorithmic and deterministic way. Meaning that, the module is about giving instruction, in this case, to a computer so that it only has one 'choice' at all times.

This term is used to describe the processes that can be used to develop solutions for complex problems. It uses a number of approaches, such as:

  • Decompose and analyse a problem
  • Design and refine solutions for a problem
  • Present these solutions in ways that can effectively be carried out by machines or humans
  • Analyse and compare these solutions

This subject should not be seen as a unit on coding/programming but rather, a unit teaching how to think in simple steps. The contents from this can be applied to more than just code development. Some of the Intended Learning Outcomes (ILOs) include:
  • The ability to decompose a problem into a series of ordered steps
  • Apply techniques such as decomposition, abstraction and generalisation to problems
  • Describe computational complexity in an informal way
  • Explain how self-referential problems relate to the limits of computation

Format

The course is run in a series of 6 blocks, each of them starting with a workshop and lasting two weeks. This is complemented by asynchronous content such as videos and additional reading.

Assessment

  • End of semester exam - 50%
  • Group coursework - 40%
    • Blocks 2-5 require a submitted solution, each of these will count 10%
    • Assessment for Block 1 is formative only so take this opportunity to learn and improve your next solutions
